What is this setup?

A personal AI operating system on a Mac Mini (MMM4). OpenClaw receives messages, applies policy, routes to the right session, calls tools, and replies. Opus orchestrates, Codex does heavy lifting, Sonnet for balanced Anthropic tasks, Haiku for cheap/fast jobs.

Flow: Input channels → Gateway → Main session → Sub-agents/tools → Memory + external systems → Output channels. Memory: session context (temporary) + durable files (MEMORY.md, memory/*.md). Notion is the external objective system; writes are manual-only.

System Architecture
Message Flow
  1. Message arrives via Slack or Webchat
  2. Gateway routes to main session
  3. Claw (Opus) reads context + memory, decides action
  4. Complex work → sub-agent with appropriate model tier
  5. Sub-agent runs, returns structured output
  6. Claw synthesizes and replies; durable decisions → memory.md
