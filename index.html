What is OpenClaw?

OpenClaw is the Gateway that runs on your Mac Mini (MMM4). It connects your AI agents to messaging channels (Telegram, iMessage, Discord), tools (web, files, GitHub, Notion), and your workspace. You talk to Claw via any of those channels ‚Äî the gateway routes it to the right agent session.

Think of it as: you ‚Üí channel ‚Üí gateway ‚Üí agent session ‚Üí tools ‚Üí reply back to you.

What is a Session?

A session is one continuous conversation thread between you and an agent. Every message you send goes into the session's context window. The agent remembers everything in the current session, but starts fresh in a new session (unless memory.md has durable facts).

Session Types ‚Äî Visual
Session Lifecycle
  1. You send a message on any channel
  2. Gateway routes it to main session
  3. Main agent decides: handle directly or spawn sub-agent
  4. If sub-agent: spawned with narrowly scoped task
  5. Sub-agent completes ‚Üí result returned to main
  6. Main synthesizes ‚Üí replies to you
  7. Durable facts written to memory.md

How to Change Config Safely
  1. Tell Claw what you want to change.
  2. Claw describes the exact change and waits for your explicit GO.
  3. Claw applies via config.patch (safe merge ‚Äî not a full overwrite).
  4. Gateway restarts automatically; pings you when back online.
  5. Durable changes also written to memory.md.

Daily Operating Pattern
  1. Check Overview: mode, posture, open loops.
  2. Pick objective ‚Üí attach one measurable next action.
